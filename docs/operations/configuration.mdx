# Configuration reference

Configuration is managed by **Spring Cloud Config Server** (`config-server` module, port 8888). All shared and service-specific properties live in the `config-repo/` directory at the monorepo root. Services fetch their merged config on startup — no shared properties are hardcoded in individual service `application.yml` files.

## Config layers

| File | Scope |
|---|---|
| `config-repo/application.yml` | Shared by **all** services — Eureka, Kafka bootstrap, Keycloak issuer-URI, JPA dialect, management endpoints, tracing, Feign timeouts, log levels, TTL settings |
| `config-repo/{service-name}.yml` | Service-specific — datasource, Redis, Kafka serializers/consumer groups, Resilience4j, MinIO, mail, CORS, gateway routes |

The Config Server merges these two files and serves the result. A service's local `src/main/resources/application.yml` contains only `spring.application.name` and the `spring.config.import` pointer:

```yaml
spring:
  application:
    name: task-service
  config:
    import: "optional:configserver:http://${CONFIG_SERVER_URL:localhost:8888}"
```

The `optional:` prefix allows services to start in isolation (e.g. unit tests) without Config Server.

Secrets and environment-specific values are injected via environment variables or Docker Compose environment blocks and referenced in `config-repo` files as `${ENV_VAR:default}`.

## Common settings (all services)

```yaml
spring:
  security:
    oauth2:
      resourceserver:
        jwt:
          issuer-uri: ${KEYCLOAK_ISSUER_URI:http://keycloak:8180/realms/demo}

management:
  tracing:
    sampling:
      probability: 1.0   # Reduce to 0.05–0.1 in production

logging:
  level:
    com.demo.common.web: DEBUG   # Controller input parameter tracing
    org.hibernate.SQL: DEBUG     # SQL logging (structured, not show-sql)
```

<Warning>
Never use `spring.jpa.show-sql: true`. It logs at INFO level with unstructured output. Use `logging.level.org.hibernate.SQL: DEBUG` instead.
</Warning>

## Eureka registration

```yaml
eureka:
  client:
    serviceUrl:
      defaultZone: http://eureka-peer1:8761/eureka/,http://eureka-peer2:8762/eureka/
  instance:
    prefer-ip-address: true
```

## Kafka

```yaml
spring:
  kafka:
    bootstrap-servers: ${KAFKA_BOOTSTRAP_SERVERS:localhost:9092}
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
    consumer:
      group-id: ${spring.application.name}
      auto-offset-reset: earliest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.JsonDeserializer
      properties:
        spring.json.trusted.packages: "com.demo.*"
```

## Redis (caching)

```yaml
spring:
  data:
    redis:
      host: ${REDIS_HOST:localhost}
      port: ${REDIS_PORT:6379}
  cache:
    type: redis
    redis:
      time-to-live: 300000   # 5 minutes (milliseconds)
```

## MinIO (file-service)

```yaml
minio:
  endpoint: ${MINIO_ENDPOINT:http://localhost:9000}
  access-key: ${MINIO_ACCESS_KEY:minioadmin}
  secret-key: ${MINIO_SECRET_KEY:minioadmin}
  buckets:
    avatars:
      allowed-types:
        - image/jpeg
        - image/png
        - image/gif
        - image/webp
      max-size-bytes: 5242880   # 5 MB
    attachments:
      allowed-types:
        - application/pdf
        - image/jpeg
        - image/png
        - application/msword
        - text/plain
        - text/csv
      max-size-bytes: 20971520   # 20 MB
```

To add a new bucket, add a new entry under `minio.buckets` in `config-repo/file-service.yml`. No Java changes required.

## API Gateway rate limits

Rate limits are configured in `config-repo/api-gateway.yml`:

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service-get
          uri: lb://USER-SERVICE
          predicates:
            - Path=/api/v1/users/**
            - Method=GET
          filters:
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 1
                redis-rate-limiter.burstCapacity: 60
                key-resolver: "#{@userKeyResolver}"
```

The `userKeyResolver` extracts the `sub` claim from the JWT, so limits are per authenticated user.

## Keycloak service accounts

Services using client credentials flow configure:

```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          keycloak:
            client-id: ${KEYCLOAK_CLIENT_ID}
            client-secret: ${KEYCLOAK_CLIENT_SECRET}
            authorization-grant-type: client_credentials
        provider:
          keycloak:
            token-uri: http://keycloak:8180/realms/demo/protocol/openid-connect/token
```

## Logstash profile

When `logstash` profile is active, a TCP appender is added:

```yaml
# application-logstash.yml
logging:
  config: classpath:logback-logstash.xml
```

The Logback config adds a `LogstashTcpSocketAppender` targeting `logstash:5000`.

## TTL & archiving

All TTL thresholds live in `config-repo/application.yml` under the `ttl` prefix. No Java changes are needed to adjust them — update the value and restart the affected service.

```yaml
ttl:
  task:
    archive-after-closed-days: 90    # days after RELEASED/REJECTED before task is archived
    archive-retention-months: 12     # months to keep archive tables before dropping them
    batch-size: 100                  # max tasks archived per nightly scheduler run
  outbox:
    retention-days: 30               # days to keep published outbox_events rows
  task-code-job:
    retention-days: 30               # days to keep processed task_code_jobs rows
  audit:
    archive-retention-months: 24     # months to keep audit archive tables
  notification:
    retention-days: 365              # days to keep notification records
  file:
    deleted-object-retention-days: 30  # days after soft-delete before MinIO object is purged
  kafka:
    task-events-retention-hours: 168   # Kafka topic retention for task-events (7 days)
    task-changed-retention-hours: 168  # Kafka topic retention for task-changed (7 days)
  elasticsearch:
    log-retention-days: 30           # days to retain Logstash log indices via ILM policy
```

<Tip>
To trigger archiving immediately in a dev environment, set `ttl.task.archive-after-closed-days: 0` and restart task-service. Revert to `90` after testing.
</Tip>

## Environment variables reference

| Variable | Default | Used by |
|---|---|---|
| `CONFIG_SERVER_URL` | `localhost:8888` | All services (config import URL) |
| `CONFIG_REPO_PATH` | `./config-repo` | config-server (filesystem backend path) |
| `KEYCLOAK_ISSUER_URI` | `http://keycloak:8180/realms/demo` | All services |
| `KAFKA_BOOTSTRAP_SERVERS` | `localhost:9092` | All services with Kafka |
| `REDIS_HOST` | `localhost` | task-service, api-gateway, user-service |
| `MINIO_ENDPOINT` | `http://localhost:9000` | file-service |
| `MINIO_ACCESS_KEY` | `minioadmin` | file-service |
| `MINIO_SECRET_KEY` | `minioadmin` | file-service |
| `KEYCLOAK_CLIENT_ID` | varies | user-service, notification-service |
| `KEYCLOAK_CLIENT_SECRET` | varies | user-service, notification-service |
| `SPRING_DATASOURCE_URL` | varies per service | All services with DB |
