# Quickstart

This guide starts all infrastructure containers and microservices on your local machine.

## Prerequisites

- **Docker Desktop** 4.x or later (with Compose v2)
- **Java 17** (Temurin or Zulu recommended)
- **Maven 3.9+**
- **Node.js 20+** and **npm**
- **Python 3.10+** (optional, for seed data script)

## Start the stack

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <repo-url>
    cd task-managment
    ```
  </Step>
  <Step title="Start infrastructure and all services">
    ```bash
    ./scripts/start-dev.sh
    ```

    This script:
    - Starts all Docker containers (Postgres, Kafka, Keycloak, Elasticsearch, Redis, MinIO, MailHog, Prometheus, Grafana, ELK)
    - Builds and starts all Spring Boot microservices
    - Opens a Terminal window per service on macOS

    To start only Docker infrastructure without microservices:
    ```bash
    ./scripts/start-dev.sh --docker-only
    ```
  </Step>
  <Step title="Wait for services to register">
    Services register with Eureka on startup. Check the Eureka dashboard:

    - Eureka peer 1: [http://localhost:8761](http://localhost:8761)
    - Eureka peer 2: [http://localhost:8762](http://localhost:8762)

    All 7 application services should appear within 30–60 seconds.
  </Step>
  <Step title="Open the frontend">
    The React SPA is served by Nginx at [http://localhost:3000](http://localhost:3000).

    Log in with one of the pre-seeded demo accounts:

    | Username | Password | Role |
    |---|---|---|
    | `admin` | `admin` | ADMIN |
    | `developer` | `developer` | DEVELOPER |
    | `qa` | `qa` | QA |
    | `pm` | `pm` | PM |
    | `supervisor` | `supervisor` | SUPERVISOR |
  </Step>
</Steps>

## Seed test data (optional)

```bash
python3 scripts/seed_task_data.py
```

Creates sample projects, tasks, and work logs for testing.

## Stop the stack

```bash
./scripts/stop-dev.sh
```

## Reset all data

<Warning>
This truncates all databases. All tasks, projects, users, and audit records will be permanently deleted.
</Warning>

```bash
./scripts/reset-data.sh
```
